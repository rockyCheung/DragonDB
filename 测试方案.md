## DragonDB & DragonStore 测试方案

### 1. 测试目标
- 验证 DragonStore 存储引擎的核心功能：键值读写、WAL 恢复、合并、缓存、Bloom 过滤等。
- 验证 DragonDB 分布式特性：文档操作、Quorum 一致性、版本向量冲突处理、集群元数据管理。
- 确保 API 层正确响应，并处理边界条件和错误。
- 保证系统在并发、故障情况下的正确性和稳定性。

### 2. 测试策略
采用分层测试，从底层到上层逐步验证：

- **单元测试**：针对单个类/函数，mock 外部依赖，快速验证逻辑正确性。
- **集成测试**：组合多个模块，测试真实交互，例如存储引擎与 WAL、SSTable 的实际读写，协调器与集群、存储的配合。
- **系统测试**：启动多个完整节点，模拟网络，测试分布式功能如数据分片、复制、故障恢复。
- **性能测试**：基准测试读写吞吐量、合并效率、延迟等。

### 3. 测试环境与工具
- **测试框架**：`pytest`（支持 asyncio），使用 `pytest-asyncio` 插件。
- **Mock**：`unittest.mock` 或 `pytest-mock` 模拟网络调用、文件系统。
- **临时目录**：`tempfile.TemporaryDirectory` 用于存储引擎的测试文件。
- **并发测试**：`asyncio` 事件循环控制，可使用 `pytest.mark.asyncio`。
- **代码覆盖率**：`pytest-cov` 生成报告。
- **性能测试**：`pytest-benchmark` 或自定义脚本。

### 5. 测试用例设计

#### 5.1 DragonStore 单元测试

| 模块 | 测试内容 |
|------|----------|
| `memtable.py` | - 插入、更新、删除键值对<br>- 查找存在/不存在/已删除的键<br>- 跳表排序和范围迭代<br>- 并发写入（通过 asyncio.gather 模拟） |
| `wal.py` | - 追加单条/批量操作<br>- 同步/异步刷盘<br>- 回放恢复（模拟崩溃后读取）<br>- 日志截断处理 |
| `cache.py` | - LRU 缓存插入和淘汰<br>- 命中/未命中<br>- 容量限制 |
| `bloom.py` | - 添加键后必定包含<br>- 假阳性率控制在预期内（大量随机键测试）<br>- 序列化与反序列化正确性 |
| `sstable/writer.py` | - 写入单个/多个数据块<br>- 索引和 Bloom 过滤器生成<br>- Footer 正确写入 |
| `sstable/reader.py` | - 点查询（存在/不存在）<br>- 利用 Bloom 过滤器提前返回<br>- 读取索引和元数据<br>- 遍历所有键（迭代器） |
| `compaction.py` | - 合并策略判断（Level0 阈值、大小阈值）<br>- 合并管理器触发和防并发<br>- 多路归并正确性（输入有序，输出有序）<br>- 合并后文件替换和清理 |
| `engine_store.py` | - put/get/delete 基本流程<br>- WAL 与 MemTable 原子性<br>- 落盘触发和 immutable 切换<br>- 从 SSTable 读取（含缓存）<br>- 关闭时落盘<br>- 崩溃恢复（模拟进程退出后重启） |

#### 5.2 DragonDB 组件单元测试

| 模块 | 测试内容 |
|------|----------|
| `coordinator/version.py` | - 向量时钟比较函数（before/after/conflict/equal）<br>- 递增版本<br>- 合并多个版本（LWW 或自定义） |
| `coordinator/coordinator.py` | - 计算分区和副本列表（mock cluster）<br>- 写操作并发调用存储（mock storage）<br>- Quorum 成功/失败处理<br>- 读操作合并版本<br>- 冲突检测与读取修复 |
| `cluster/hashring.py` | - 添加/移除节点后虚拟节点分布<br>- 同一个 key 始终映射到相同节点<br>- 节点变化后少量 key 迁移 |
| `cluster/manager.py` | - 获取副本列表（顺序和数量）<br>- 节点在线状态管理<br>- 分区映射更新 |
| `api/handlers.py` | - 请求解析和参数验证<br>- 调用 coordinator 并返回正确 HTTP 状态码<br>- 错误处理（如文档不存在返回 404） |

#### 5.3 集成测试

- **存储引擎内部集成**：模拟连续写入触发落盘和合并，验证数据最终正确性（通过读取所有键）。例如：写入 10 万条数据，等待合并，重启引擎，读取所有键确认未丢失。
- **协调器+存储**：使用内存存储模拟，测试协调器的 Quorum 逻辑，验证版本向量在多个副本间的维护。
- **集群+协调器**：启动多个 `DragonDBNode` 实例（内存存储），测试分区和副本分配，写入后从不同节点读取。

#### 5.4 系统测试（多节点）

- **环境**：使用 `pytest-xdist` 或 `subprocess` 启动多个进程，每个节点监听不同端口，使用临时目录存储。
- **测试场景**：
  - 数据分片：写入 1000 个文档，验证它们均匀分布在各个节点。
  - 复制与一致性：设置 N=3, W=2, R=2，写入后关闭一个节点，读取仍能成功（quorum）。
  - 故障恢复：节点重启后数据重新同步（需实现读取修复或 hinted handoff，当前设计可能未包含，可简化测试）。
  - 并发冲突：同时更新同一文档，验证向量时钟检测并返回冲突。

### 7. 测试代码示例

#### 7.1 测试 memtable 插入和查找（`test_memtable.py`）

```python
import pytest
from dragondb.storage.dragonstore.memtable import SkipList

def test_put_and_get():
    sl = SkipList()
    key = b'foo'
    value = b'bar'
    sl.put(key, value)
    assert sl.get(key) == value

def test_get_non_existent():
    sl = SkipList()
    assert sl.get(b'nonexist') is None

def test_delete():
    sl = SkipList()
    sl.put(b'k1', b'v1')
    sl.delete(b'k1')
    assert sl.get(b'k1') is None
    assert len(sl) == 0

def test_items_order():
    sl = SkipList()
    pairs = [(b'a', b'1'), (b'b', b'2'), (b'c', b'3')]
    for k, v in pairs:
        sl.put(k, v)
    items = sl.items()
    assert items == pairs
```

#### 7.2 测试 WAL 恢复（`test_wal.py`）

```python
import pytest
import asyncio
from dragondb.storage.dragonstore.wal import WAL

@pytest.mark.asyncio
async def test_wal_append_and_replay(temp_dir):
    wal_path = temp_dir / 'test.log'
    wal = WAL(str(wal_path))
    await wal.open()
    ops = [('put', b'k1', b'v1'), ('put', b'k2', b'v2'), ('delete', b'k1', None)]
    await wal.append(ops)
    await wal.close()

    # 重新打开回放
    wal2 = WAL(str(wal_path))
    await wal2.open()
    replayed = await wal2.replay()
    await wal2.close()
    assert replayed == ops
```

#### 7.3 测试协调器写操作（`test_coordinator.py`）

```python
import pytest
from unittest.mock import AsyncMock, Mock
from dragondb.coordinator.coordinator import Coordinator
from dragondb.models.document import Document

@pytest.mark.asyncio
async def test_write_success(mock_cluster_manager):
    # 准备 mock 存储
    storage = AsyncMock()
    storage.put.return_value = True
    coord = Coordinator('node1', storage, mock_cluster_manager)
    # 注册远程存储（这里用同一个 storage 模拟）
    coord.register_remote('node1', storage)
    coord.register_remote('node2', storage)
    coord.register_remote('node3', storage)

    result = await coord.write('users', '1001', {'name': 'Alice'}, w=2)
    assert result['status'] == 'success'
    assert storage.put.call_count == 3  # 三个副本都被调用
```

### 8. 持续集成与报告
- 使用 GitHub Actions 或 GitLab CI 运行测试。
- 配置 `pytest --cov=dragondb --cov-report=html` 生成覆盖率报告。
- 使用 `flake8` 或 `black` 进行代码风格检查。

### 9. 注意事项
- 由于 DragonStore 涉及文件 I/O，测试时应使用临时目录，并确保清理。
- 异步测试需使用 `pytest.mark.asyncio`，并注意事件循环管理。
- 系统测试可能需要较长运行时间，可标记为 `pytest.mark.slow` 选择性执行。

此方案覆盖了从单元到系统的各个层面，确保 DragonDB 和 DragonStore 的质量和可靠性。
